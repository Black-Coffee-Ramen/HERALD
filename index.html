Detection Capabilities

30+ engineered features.
One ensemble.

Our ML ensemble evaluates every candidate across lexical, visual, DNS, and behavioural dimensions simultaneously — catching attacks at every evasion layer.

🔤
Typosquatting Detection
Edit distance analysis, keyboard adjacency scoring, and character substitution patterns catch domains engineered to fool human eyes.
sbi→5bihdfc→hdfc1edit-distance
🌐
IDN / Homoglyphs
Unicode confusable character detection across Cyrillic, Greek, and Latin scripts. Catches domains that look identical in browser address bars.
unicodecyrillicgreek-sub
🔍
Fuzzy Brand Matching
Levenshtein distance scoring against a configurable CSE keyword list. Catches subtle variants like uldai, hdfc1, 5bi that exact matching misses.
levenshteinfuzzy-matchconfigurable
📁
Path-Based Phishing
Detects brand keywords buried in URL paths on generic domains — a classic evasion technique missed by domain-only scanners entirely.
url-pathkeyword-scanevasion
⚠️
TLD Risk Scoring
Explicit penalty weights for high-risk gTLDs: .xyz, .top, .buzz, .tk, .ml and others. Composite scoring with domain-level signals for precision.
.xyz.top.buzz.tk
🚇
Tunnel Detection
Flags Ngrok, Vercel, Cloudflare Tunnel subdomains serving lookalike content — attackers increasingly use tunnelling to bypass traditional blocklists.
ngrokvercelcf-tunnel
👁️
Visual Similarity
OCR + perceptual hashing compares live screenshots against stored CSE page templates. Catches phishing with zero URL resemblance to target brand.
ocrperceptual-hashchromium
📜
SSL Analysis
Certificate age, issuer, Subject Alternative Names, and mismatch detection. Fresh Let's Encrypt certs on brand-name domains are a strong signal.
cert-agesan-checkissuer
🕐
Domain Age & WHOIS
Registration recency and WHOIS history correlation. Freshly registered domains with brand keywords are an immediate high-confidence phishing signal.
whoisregistrarreg-date

Self-Contained Architecture

Five services.
One docker compose up.

Everything runs in a single Docker network. PostgreSQL for domain history. Redis for the async scan queue. No external dependencies beyond public data sources.

Streamlit Dashboard
Live alert feed with CSE↔phishing mapping. Domain enrichment viewer. Suspected domain queue management. Streamlit UI for rapid deployment.
:8501 · http://localhost:8501
FastAPI REST Layer
REST endpoints for domain scanning, report retrieval, and suspected domain management. Interactive Swagger docs at /docs. Full OpenAPI spec included.
:8000 · http://localhost:8000/docs
Background Workers
Certstream monitor, hourly domain poller, Telegram scraper, and APScheduler for the suspected domain re-check queue. All persistent background processes.
Internal · no port exposed
PostgreSQL
Primary data store for domain history, scan records, WHOIS enrichment data, and the suspected domain queue. SQLAlchemy ORM models.
:5432 · Internal only
Redis
Async job queue for scan requests submitted via API. Decouples the FastAPI request cycle from heavyweight ML inference and screenshot pipeline.
:6379 · Internal only
Docker Network
All services isolated in a single Docker network. Only ports 8000 and 8501 exposed to host.
Quick Start

Up and running
in one command.

bash
# Clone the repository
git clone github.com/Black-Coffee-Ramen/HERALD
cd HERALD

# Start all services (recommended)
docker compose up --build

# Requires Docker with 8GB+ RAM allocated

# Dashboard: http://localhost:8501
# API docs: http://localhost:8000/docs
python3.12 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# Install ChromeDriver for screenshots
sudo apt install -y chromium-chromedriver

# Train model on your dataset
python train_model.py --training_data data/training/

streamlit run app/dashboard.py

API Reference

FastAPI REST layer.
Full Swagger docs included.

Integrate HERALD directly into your existing infrastructure. Interactive docs at localhost:8000/docs when running.

POST/api/scan
Submit any domain for immediate ML scanning. Returns classification, confidence score, and full enrichment data in the response.
POST /api/scan
{"domain": "sbi-login-secure.xyz"}

→ classification, score, whois, ip, asn
GET/api/suspected
Retrieve all currently monitored suspected domains — parked domains flagged for re-monitoring with scores and registration timestamps.
GET /api/suspected

→ [{domain, score, registered, last_checked}]
GET/api/report/{domain}
Full enrichment report for any domain. Returns WHOIS, IP, ASN, MX records, SSL certificate details, screenshot URL, and all triggered feature flags.
GET /api/report/sbi-login-secure.xyz

→ whois, ip, asn, ssl, mx, screenshot_url, flags
GET/api/health
System health check — returns service status for all workers, queue depth, certstream connection status, and system uptime.
GET /api/health

→ {status, queue_depth, certstream, uptime}
